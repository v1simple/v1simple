# Changelog

All notable changes to this project are documented here.

## [4.2.20] — 2026-06-19

### Security

- Require the bundled maintenance UI's `X-V1Simple-Request: maintenance-ui`
  header on mutating `POST /api/*` requests, blocking ordinary cross-origin
  form posts from changing device state.
- Add a contract gate so future mutating API routes must include the same
  maintenance write guard.
- Replace generic LittleFS fallback serving with an explicit allowlist for
  shipped HTML pages, Svelte assets, branding images, and voice prompt assets.

### Release

- Bump firmware version metadata to 4.2.20 for the next web-installer release.

## [4.2.14] — 2026-05-25

### Performance

- Skip unchanged display work to reduce redundant pipeline flushes.
- Gate display correctness trace on relevance to avoid formatting cost on idle frames.
- Throttle tap gesture polling frequency.
- Attribute over-cap dirty regions for display pipeline observability.
- Shrink numeric frequency redraw scope.
- Right-size the BLE notification queue and perf SD writer stack from hardware evidence.
- Shorten perf snapshot critical sections and remove retired perf/display/audio telemetry columns.

### Reliability

- Guard BLE queue init: `begin()` failure now triggers a fatal boot error rather than silently continuing without a usable BLE RX path.
- Fix duplicate touchscreen tap registration after transient AXS15231B I2C read failures.
- Reset perf CSV session-window peak metrics when starting a new SD logging session.
- Split runtime wiring into `main_runtime_wiring.cpp` while preserving drive/maintenance boot policy.
- Promote the pioarduino ESP32 platform pin to the hardware-validated production toolchain.
- Harden LittleFS image generation/compatibility checks and release merged-image flash-policy validation.
- Add memory-headroom, release-workflow, frontend-resilience, API-source, and module-source contract gates.
- Add PR CI coverage plus an advisory native-sanitized unit-test lane.
- Add unit coverage for WiFi priority policy, qualification serial bench control, and production WiFi manager symbols.
- Publish the ESP Web Tools installer to GitHub Pages from the same manifest and merged firmware image attached to each release.

### Cleanup

- Remove local audio playback, retired audio settings/perf surfaces, and legacy WiFi control routes.
- Remove stale display logging surfaces, dead boot-logo conversion tooling, and redundant packet-parser global reset state.
- Document the private web-installer template ownership and keep release assets generated by the release workflow.
- Refresh ALP/V1 laser-handoff wording to avoid retired ALP terminology in comments and UI copy.

### Terminal release notes

- Publish this as a single-root public snapshot; earlier development lineage is intentionally not represented in `main` history.
- Record physical LCD pixel-layer checks (`C-PERSIST-HOLD`, `C-BLINK-CADENCE`, `C-BENCH-EYES`) as manual or aspirational, not automated.
- Ship OBD/proxy paths under an explicit `ACCEPTED_RISK` release waiver because no representative OBD/proxy hardware qualification rig exists.
- Keep the release workflow on verified current GitHub Actions major tags, including `actions/checkout@v6` and `actions/setup-node@v6`.
